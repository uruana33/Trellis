## README 中的 Use Case

**BMAD-METHOD README 没有专门的 "Use Cases" 章节**，但通过**三种轨道**和**四阶段模型**隐含展示。

### README 中的轨道系统 Use Case

| 轨道 | 用途 | 时间 |
|------|------|------|
| **Quick Flow** | Bug 修复、小功能、快速迭代 | ~5 分钟 |
| **BMad Method** | 产品与平台开发、中型项目 | ~15 分钟 |
| **Enterprise** | 合规性强的系统、大型项目 | ~30 分钟 |

### README 中的四阶段模型 Use Case

| 阶段 | Use Case |
|------|----------|
| Phase 1 (Analysis) | 可选探索，理解问题域 |
| Phase 2 (Planning) | PRD 创建，定义需求 |
| Phase 3 (Solutioning) | 架构设计，技术方案 |
| Phase 4 (Implementation) | 迭代开发，代码实现 |

### README 中的核心特性 Use Case

| 特性 | 隐含 Use Case |
|------|--------------|
| Scale-Adaptive Intelligence | 自动适应项目复杂度（0-4 级） |
| Structured Workflows | 50+ 引导工作流，规范化开发 |
| Specialized Agents | 21 个专业代理，分工协作 |
| Complete Lifecycle | 完整开发生命周期管理 |

### README 中的三个模块 Use Case

| 模块 | Use Case |
|------|----------|
| BMad Method (BMM) | 核心敏捷开发框架 |
| BMad Builder (BMB) | 自定义代理和模块创建 |
| Creative Intelligence Suite (CIS) | 创意、头脑风暴、问题解决 |

### Use Case 展示风格
- **轨道分类**：按项目规模和复杂度分类
- **阶段模型**：四阶段流程展示完整开发周期
- **表格驱动**：大量使用表格对比不同选项
- **无具体案例**：没有"用 BMAD 做了 X 项目"的描述
