# Version 3 设计说明：实战演示风格

> 风格定位：**"Show, Don't Tell"**

---

## 设计理念

**参考竞品：** Aider、OpenCode（代码密集、示例驱动）

**核心原则：**
- 用实际输出展示功能，而非描述功能
- 模拟真实终端交互
- 让用户"看到"而非"读到"

---

## 风格特点

| 维度 | Version 3 |
|------|-----------|
| **行数** | ~280 行 |
| **标语** | 问题陈述（"Your AI assistant forgets everything. Trellis remembers."） |
| **开篇** | 3 个完整的交互演示（Session 1, Session 2, Multi-Agent） |
| **解释** | 最小化，让代码说话 |
| **表格** | 命令速查表（实用导向） |
| **代码块** | 大量（占 60%+ 篇幅） |

---

## 核心亮点

### 1. 模拟终端对话

```
你:   /start
AI:   ✓ 读取 workflow.md
      ✓ 开发者: alice
      准备就绪。我们要做什么？
```

**效果：** 用户立即理解"用起来是什么感觉"

### 2. 流水线可视化

```
┌─ Plan Agent ─────────────────────────────────────┐
│ ✓ 分析代码库...                                  │
│ ✓ 创建功能目录: 19-user-auth/                    │
└──────────────────────────────────────────────────┘
```

**效果：** 复杂流程一目了然

### 3. 真实配置示例

不是"配置项说明"，而是"真实可用的配置文件"：

```yaml
worktree_dir: ../trellis-worktrees
copy:
  - .env
  - .trellis/.developer
verify:
  - pnpm lint
  - pnpm typecheck
```

---

## 适用场景

- 目标用户是动手型开发者
- 产品功能复杂，需要展示"实际效果"
- 没有视频/GIF 演示时的替代方案
- 强调"眼见为实"

---

## 借鉴来源

| 竞品 | 特点 |
|------|------|
| **Aider** | 大量终端输出示例，展示实际对话 |
| **OpenCode** | 命令输出示例，强调实用性 |
| **Planning-with-Files** | 展示文件内容结构 |

---

## 优点

1. 用户无需想象，直接看到效果
2. 降低认知门槛（"原来是这样用的"）
3. 可直接复制粘贴配置
4. FAQ 解决常见疑虑

## 缺点

1. 代码块占用大量空间
2. 不熟悉终端的用户可能困惑
3. 需要保持示例与实际功能同步
4. 翻译难度大（输出也需要翻译）
