# Version 2 设计说明：极简风格

> 风格定位：**"Less is More"**

---

## 设计理念

**参考竞品：** Cursor（README 极短，一切指向文档站）

**核心原则：**
- README 只是入口，不是文档
- 能少说就少说
- 一切细节去 docs

---

## 风格特点

| 维度 | Version 2 |
|------|-----------|
| **行数** | ~70 行 |
| **标语** | 1 句话（"Workflow templates for AI coding assistants"） |
| **结构** | Logo → 安装 → 使用 → 目录 → 链接 |
| **解释** | 几乎没有 |
| **表格** | 无 |
| **代码块** | 仅安装和命令 |

---

## 适用场景

- 已有独立文档站
- 用户群体是快速上手型开发者
- 品牌认知度高，不需要长篇介绍
- 强调"先装上试试"

---

## 借鉴来源

| 竞品 | 特点 |
|------|------|
| **Cursor** | README 极短，只有几行，一切指向官网 |
| **OpenSpec** | 简洁入口，详细内容在 thedocs.io |

---

## 优点

1. 首屏即全部，无需滚动
2. 安装命令一眼可见
3. 不会吓跑新手（信息量小）
4. 维护成本低

## 缺点

1. 无法展示项目复杂度
2. 新用户不知道"为什么要用"
3. SEO 效果差（内容少）
4. 依赖外部文档站
