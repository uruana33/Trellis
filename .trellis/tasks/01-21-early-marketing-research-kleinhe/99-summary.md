## 6 种早期营销模式

### 模式 1: YC 系统打法 (Continue)
```
YC 录取 → Show HN → 媒体报道 → 融资公告 → Product Hunt
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 最强信任背书 | 需要 YC 录取 | 即时爆发 |

### 模式 2: 品牌借势 + 播客深度 (OpenCode)
```
已有品牌(SST) → 静默发布 → 积累用户 → HN 爆发 → 播客采访
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 有时间打磨产品 | 需要已有品牌资产 | 2-3 个月 |

### 模式 3: 博客首发 + KOL 放大 (Superpowers) ⭐ 推荐
```
方法论博客 → 产品发布 → KOL 推荐 → GitHub Trending
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 低成本，内容可控 | 需要 KOL 关系或被发现 | 次日即可爆发 |

### 模式 4: 概念顺风车 + 内容矩阵 (OpenSpec)
```
抢占概念 → 等待官方验证 → 多平台内容 → 缓慢积累
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 无需资源，借势 | 需要判断概念趋势 | 3+ 个月 |

### 模式 5: 内部工具开源 + YouTube 教程矩阵 (Roo Code)
```
内部痛点 → Fork 改进 → 开源 → 时机事件 → YouTube 教程矩阵
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 产品已验证，真实需求 | 需要内部使用场景 | 事件触发后快速 |

### 模式 6: 纯产品驱动 + 被动流量入口 (claude-mem) 🆕
```
强痛点产品 → GitHub 发布 → 极速迭代 → 被动入口发现 → 第三方自发报道
```
| 优点 | 缺点 | 时间线 |
|------|------|--------|
| 零营销成本，专注产品 | 需要强痛点，增长较慢 | 4+ 个月 |

**关键要素**: 痛点足够强 + 极高迭代频率 (168版本) + 被动入口 (awesome list, daily.dev, marketplace)

## 核心真相

### 1. KOL 推荐 > 渠道曝光
- Superpowers 没有 HN/PH，靠 Simon Willison **一条推荐**爆发
- **一个高质量 KOL > 100 条普通推广**

### 2. 方法论内容 > 产品公告
- obra 的博客不是"我做了个工具"
- 而是"这是我如何用 AI 编码"
- **教育市场的同时建立权威**

### 3. 时机窗口 > 一切
- Superpowers 在 Anthropic 发布插件系统**当天发布**
- OpenSpec 在 GitHub Spec Kit **之前启动**，借势爆发
- Roo Code 在 DeepSeek R1 发布后**立即集成**

### 4. 创始人品牌是长期资产
- Dax: 73K Twitter 粉丝 → 初始流量池
- obra: Wikipedia 词条 → 信任背书

### 5. YouTube 是被低估的增长渠道
- Roo Code **没有 HN 首发爆发**
- 但通过 YouTube 教程矩阵实现 21.8K stars + 1.16M installs

## Trellis 行动建议

### 最高优先级 (立即行动)

#### 1. 找到你的 Simon Willison
**目标 KOL 清单**:
| KOL | 平台 | 影响力 | 接触策略 |
|-----|------|--------|----------|
| Simon Willison | Blog/Mastodon | 极高 | 发送产品让其自然发现 |
| Latent Space | Podcast | 高 | 争取播客采访机会 |
| Better Stack | YouTube | 中 | 主动联系评测 |
| Kent C. Dodds | Podcast | 高 | 技术采访 |

**行动步骤**:
1. 列出 10 个 AI Coding 领域活跃 KOL
2. 关注他们的内容，理解他们的兴趣点
3. 在产品成熟时，发送个性化邀请体验

#### 2. 方法论博客先行
**模仿 Superpowers 的策略**:
1. **先写方法论文章** - "我如何使用 AI Coding Agent 管理复杂项目"
2. **不提产品，只讲方法** - 建立"这个人懂行"的信任
3. **产品作为方法论的实现** - 自然过渡，不像广告
4. **博客首发，多平台分发** - 不急于 HN/PH

#### 3. 概念绑定
**Trellis 需要绑定的概念** (参考):
- "Governance Framework for AI Coding"
- "Session-based AI Development"
- "AI Workflow Traceability"
- "Project-Aware AI Coding"

**策略**:
- 在所有内容中反复使用同一概念
- 让这个概念成为品类名称

### 高优先级

#### 4. HN Show/Launch
- 2/5 产品靠 HN 首发爆发
- 300 points 是门槛
- 时机: 工作日上午 (美国时间)
- **建议**: 先博客首发积累口碑，再 HN

#### 5. 创始人品牌建设
- LinkedIn 定期分享思考
- 技术博客建立专业形象
- 个人品牌是长期资产

### 中优先级

#### 6. 播客/采访
- Latent Space, Chats with Kent 等
- 长内容建立思想领导力
- 不是广告，是观点分享

#### 7. 内容矩阵
- Medium, DEV.to, YouTube, LinkedIn
- 不同平台不同调性
- 量变产生质变

#### 8. YouTube 教程生态
- 让产品易于制作教程:
  - 清晰的安装步骤
  - 可展示的视觉效果
  - "免费使用" 叙事吸引预算敏感用户

## 推荐首发策略

### Superpowers 模式 (博客首发 + KOL 放大)

```
Week -2: 方法论博客文章 (不提产品)
Week -1: 产品打磨，准备 demo
Day 0:   产品博客发布 + LinkedIn 分享
Day 1:   联系 KOL (如已建立关系)
Day 7:   根据反馈优化
Week 4:  HN Show 首发 (如 KOL 放大效果一般)
```

**为什么不推荐直接 HN 首发**:
- HN 只有一次机会
- 产品未经验证可能反馈差
- KOL 放大如果成功，HN 可以锦上添花

### 时机窗口关注
- Anthropic/OpenAI 的重大发布
- GitHub 的新功能发布
- 行业热点事件 (新模型发布)
